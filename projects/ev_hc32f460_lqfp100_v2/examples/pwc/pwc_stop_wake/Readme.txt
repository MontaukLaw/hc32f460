平台说明
================================================================================
GCC工程，由Eclipse IDE外挂GNU-ARM Toolchain，再结合pyOCD GDB Server实现工程的编译、
链接和调试。在用Eclipse导入工程后，请将xxxx_PyOCDDebug中pyocd-gdbserver和SVD文件
设置为正确的路径；请将xxxx_PyOCDDownload中pyocd设置为正确的路径。注意，这些路径不
能包含非英文字符。

================================================================================
功能描述
================================================================================
本样例为芯片进入停止模式并通过KEY10唤醒，分别以PWC_STOP_WFI,PWC_STOP_WFE_INT,
PWC_STOP_WFE_EVT三种方式进入停止模式，分别对应NVIC中断，中断事件请求，事件请求三种唤醒方式。

================================================================================
测试环境
================================================================================
测试用板:
---------------------
EV_F460_LQ100_Rev2.0

辅助工具:
---------------------
无

辅助软件:
---------------------
串口助手

================================================================================
使用步骤
================================================================================
1）打开工程并重新编译；
2）设置串口助手115200bps@8N1，并打开PC端所识别到的COM口；
3）启动IDE的下载和调试功能，全速运行；
4）按下按键K10，串口打印“MCU will enter stop mode by PWC_STOP_WFI...”，芯片随即进入停止模式；
5）再次按下按键K10，可唤醒芯片，蓝灯闪烁5次后，串口打印
   MCU wake up from stop mode by interrupt handle!

   MCU will enter stop mode by PWC_STOP_WFE_INT...
   芯片随即进入停止模式；
6）再次按下按键K10，可唤醒芯片，串口打印
   MCU wake up from stop mode by interrupt request!

   MCU will enter stop mode by PWC_STOP_WFE_EVT...
   芯片随即进入停止模式；
7）再次按下按键K10，可唤醒芯片，串口打印
   MCU wake up from stop mode by event!

   MCU will enter stop mode by PWC_STOP_WFI...
   芯片随即进入停止模式；
8）可重复步骤5）~7）

================================================================================
注意
================================================================================
1. 进去停止模式前，确保无DMA传输，无Flash擦写操作;
2. 此样例使芯片进入低功耗模式，若出现调试端口无法使用，请保证开发按键K10未被按下的
   情况下，复位系统板。
