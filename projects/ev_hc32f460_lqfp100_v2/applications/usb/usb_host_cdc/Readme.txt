平台说明
================================================================================
GCC工程，由Eclipse IDE外挂GNU-ARM Toolchain，再结合pyOCD GDB Server实现工程的编译、
链接和调试。在用Eclipse导入工程后，请将xxxx_PyOCDDebug中pyocd-gdbserver和SVD文件
设置为正确的路径；请将xxxx_PyOCDDownload中pyocd设置为正确的路径。注意，这些路径不
能包含非英文字符。

================================================================================
功能描述
================================================================================
本样例主要展示MCU USB外设实现的USB Host CDC功能，Host主动发送数据到Device，验证是否
可以收到Device返回的相同数据，以验证CDC通信是否正常。
（为了实现Device自动返回数据，需将Device的UART TX和RX进行短接）。

说明：测试连接示意图如下
                             host tx data --->--------------------------------┐
 +--------------+                                                             |
 |PC Serial Port|         +----------------+        +-----------------+       |
 |  Assistant   |         |      USB       |        |      USB      TX+----+  |
 |              |      XXX|DAP   Host   USB|XXXXXXXX|USB   Device     |    |  |
 +--------------+     XX  |      Board     |        |      Board    RX+----+  |
     |-----|         XX   +----------------+        +-----------------+       |
+----------------+XXXX                                                        |
+----------------+           host rx data ---<--------------------------------┘

================================================================================
使用步骤
================================================================================
1）打开工程编译并全速运行在主机开发板上，使用主机板载DAP虚拟串口查看运行输出信息；
2）J14连接OTG转接线，再接上CDC设备；
  （使用另一块HC32开发板烧录usb_dev_cdc样例获得该CDC设备）
3）设备枚举完成后，主机DAP虚拟串口提示：
   “Please short the CDC device's UART TX and RX, then press host's K1 to test”；
4）按提示短接CDC设备的串口数据TX和RX端口（端口丝印见对应的usb_dev_cdc样例说明）；
5）按下主机的K1按键，观察测试结果是否为通过：
   通过：主机开发板蓝色亮起，且DAP虚拟串口提示“CDC Test passed”；
   失败：主机开发板红灯亮起，且DAP虚拟串口提示“CDC Test failed”；
   超时：主机开发板红灯亮起，且DAP虚拟串口提示“CDC Test timeout”。

================================================================================
注意
================================================================================
1）为了保证供电充足，注意将主机板子J16 PWRSEL的DAP跳线短接；
2）CDC设备的串口数据收发使用的端口，请参考对应的usb_dev_cdc样例说明。
